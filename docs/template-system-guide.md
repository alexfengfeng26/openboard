## 一、模板系统概述

模板系统是 kanban-board 的统一内容复用引擎，将原先分散在 3 处的硬编码模板整合为一套完整的存储、管理和应用体系。

### 核心能力

| 能力 | 说明 |
|------|------|
| **5 类模板** | 看板、卡片、列表、自动化规则、AI 提示词 |
| **内置模板** | 系统预设 16+ 模板，开箱即用 |
| **自定义模板** | 基于内置模板复制，或从零创建 |
| **变量解析** | 支持 `{{variable}}` 动态替换 |
| **导入/导出** | JSON 格式，方便团队共享 |
| **作用域** | 全局模板 vs 看板级私有模板 |

### 模板生命周期

```
内置模板（只读）
    ↓ 复制
自定义模板（可编辑）
    ↓ 应用
看板/卡片/列表/规则/Prompt（业务数据）
```

## 二、模板类型详解

### 2.1 看板模板（`board`）

用于创建新看板时快速生成预设的列表结构和标签体系。

**内容结构：**

```typescript
{
  lanes: [
    { title: '待办' },
    { title: '进行中' },
    { title: '已完成' }
  ],
  tags: [
    { id: 'tag-1', name: '紧急', color: '#ef4444' }
  ]
}
```

**使用场景：**
- 新建项目看板时选择"Scrum Sprint"模板
- 内容团队创建"内容运营"看板
- 研发团队使用"Bug 跟踪"模板

---

### 2.2 卡片模板（`card`）

用于创建卡片时快速填充标题前缀、描述格式和标签。

**内容结构：**

```typescript
{
  title: '【Bug】',
  description: '**问题描述**：\n\n**复现步骤**：\n1. \n2. \n3. ',
  tags: ['Bug']
}
```

**使用场景：**
- 提交 Bug 时选择"Bug 报告"模板，自动填充标准格式
- 提需求时选择"需求申请"模板

---

### 2.3 列表模板（`lane`）

用于在看板中快速添加预设的列表，可附带初始卡片。

**内容结构：**

```typescript
{
  title: 'Review',
  cards: [
    { title: '代码审查清单', description: '- [ ] 命名规范\n- [ ] 单元测试' }
  ]
}
```

**使用场景：**
- Sprint 中期新增"Review"列表
- 项目收尾阶段添加"归档"列表

---

### 2.4 自动化规则模板（`automation`）

用于快速创建自动化规则，绑定触发器和动作。

**内容结构：**

```typescript
{
  trigger: { type: 'card_created', conditions: [] },
  actions: [
    { type: 'auto_tag', params: {} }
  ]
}
```

**使用场景：**
- 新卡片创建时自动匹配标签
- 卡片移动后自动补充分类标签

---

### 2.5 AI 提示词模板（`prompt`）

用于 DeepSeek 聊天面板中快速发送预设 Prompt，支持变量替换。

**内容结构：**

```typescript
{
  text: '帮我生成每日站会的 3 张卡片：\n1. 昨日完成\n2. 今日计划\n3. 阻塞项',
  model: 'deepseek-chat',
  autoSend: false
}
```

**使用场景：**
- 每日站会一键生成卡片
- Sprint 规划时批量创建任务卡片
- 代码审查前生成检查清单

## 三、内置模板速查

### 3.1 看板模板（4 个）

| 模板 | 用途 | 列结构 |
|------|------|--------|
| 基础看板 | 个人任务管理 | 待办 → 进行中 → 已完成 |
| Scrum Sprint | 敏捷开发 | Backlog → Todo → In Progress → Review → Done |
| 内容运营 | 内容创作流程 | 选题池 → 撰写中 → 审核中 → 已发布 → 归档 |
| Bug 跟踪 | 缺陷管理 | 新提交 → 确认中 → 修复中 → 待验证 → 已关闭 |

### 3.2 卡片模板（2 个）

| 模板 | 标题前缀 | 标签 |
|------|----------|------|
| Bug 报告 | `【Bug】` | Bug |
| 需求申请 | `【需求】` | 功能 |

### 3.3 列表模板（5 个）

| 模板 | 用途 |
|------|------|
| 待办列表 | 收集尚未开始的任务 |
| 进行中列表 | 跟踪正在处理的任务 |
| 已完成列表 | 归档已完成的任务 |
| Backlog | 沉淀待排期的需求 |
| Review | 代码审查/内容审核 |

### 3.4 自动化规则模板（3 个）

| 模板 | 触发时机 | 动作 |
|------|----------|------|
| 创建时自动匹配标签 | 卡片创建 | auto_tag |
| 更新时自动匹配标签 | 卡片更新 | auto_tag |
| 移动后自动匹配标签 | 卡片移动 | auto_tag |

### 3.5 AI 提示词模板（3 个）

| 模板 | 用途 |
|------|------|
| 每日站会 | 生成昨日完成/今日计划/阻塞项卡片 |
| Sprint 规划 | 生成本 Sprint 目标与关键任务 |
| 代码审查 | 生成规范性/逻辑性/性能检查清单 |

## 四、使用模板

### 4.1 创建看板时使用模板

**操作路径：**

1. 点击顶部工具栏的 **「+ 新建看板」**
2. 输入看板名称
3. 选择 **「从模板库选择」**
4. 在弹窗中选择模板类型（默认 `board`）
5. 点击目标模板的 **「使用」** 按钮

**示例：**

> 假设你要为一个新项目的 Sprint 1 创建看板：
>
> 1. 输入名称：`Sprint 1 - 用户系统`
> 2. 点击「从模板库选择」
> 3. 选择「Scrum Sprint」模板
> 4. 点击「使用」
> 5. 系统自动创建看板，包含 Backlog / Todo / In Progress / Review / Done 五列

---

### 4.2 创建卡片时使用模板

**操作路径：**

1. 在任意列表点击 **「+ 添加卡片」**
2. 在卡片表单中点击 **「选择模板」**
3. 选择卡片模板（如「Bug 报告」）
4. 系统自动填充标题前缀和描述格式
5. 补充具体内容后保存

**示例：**

> 发现一个登录页面崩溃的 Bug：
>
> 1. 在「Bug 跟踪」看板的「新提交」列创建卡片
> 2. 点击「选择模板」→「Bug 报告」
> 3. 标题自动变为：`【Bug】登录页面在输入特殊字符时崩溃`
> 4. 描述自动填充：
>    ```
>    **问题描述**：
>
>    **复现步骤**：
>    1. 打开登录页面
>    2. 在用户名输入框输入 emoji
>    3. 点击登录按钮
>
>    **期望结果**：
>
>    **实际结果**：
>
>    **环境信息**：
>    ```

---

### 4.3 添加列表时使用模板

**操作路径：**

1. 在看板右侧点击 **「+ 添加列表」**
2. 选择「从模板库选择」
3. 选择列表模板（如「Review」）
4. 系统自动在末尾添加该列表

---

### 4.4 使用 AI 提示词模板

**操作路径：**

1. 打开右侧 **AI 聊天面板**
2. 在输入框上方或快捷命令区域选择 Prompt 模板
3. 模板文本自动填入输入框
4. 按 Enter 发送或继续编辑

**示例：**

> 每日站会场景：
>
> 1. 打开 AI 面板
> 2. 选择「每日站会」模板
> 3. 输入框自动填入：
>    ```
>    帮我生成每日站会的 3 张卡片：
>    1. 昨日完成的工作
>    2. 今日计划
>    3. 当前的阻塞或风险
>    每张卡片包含标题和简短描述。
>    ```
> 4. 发送后 AI 会返回 3 张结构化卡片建议

## 五、复制与自定义模板

### 5.1 为什么需要复制？

内置模板（带 **「内置」** Badge）**不能直接编辑或删除**。这是为了保证系统模板的稳定性。

如果你想：
- 修改"Scrum Sprint"看板的列名（如把"Review"改成"测试中"）
- 调整"Bug 报告"卡片的描述格式
- 在"每日站会"Prompt 中添加团队特定的要求

就必须先**复制**内置模板，生成一个可自由编辑的自定义版本。

---

### 5.2 复制操作

**在选择弹窗中复制：**

```
[Scrum Sprint] [内置]
  敏捷开发 Sprint 看板...
  [使用] [复制]
            ↑
         点击「复制」
```

点击后系统自动：
1. 克隆该模板（`builtin` 变为 `false`）
2. 名称变为 `Scrum Sprint (自定义)`
3. 直接选中并应用新副本

**在模板管理页面复制：**

```
模板库 → 选择「看板」类型 → 找到目标模板
  [使用] [编辑] [删除] [复制]
                            ↑
                         点击「复制」
```

---

### 5.3 自定义模板示例

#### 示例：基于「Scrum Sprint」创建团队定制版

**步骤 1：复制模板**

在模板管理页面找到「Scrum Sprint」，点击「复制」。

**步骤 2：编辑副本**

点击新副本的「编辑」按钮，修改内容：

```json
{
  "meta": {
    "name": "Scrum Sprint (前端团队版)",
    "description": "前端团队定制 Sprint 流程，包含设计走查环节",
    "tags": ["前端", "Scrum"]
  },
  "content": {
    "lanes": [
      { "title": "Backlog" },
      { "title": "设计中" },
      { "title": "开发中" },
      { "title": "Code Review" },
      { "title": "设计走查" },
      { "title": "Done" }
    ],
    "tags": [
      { "id": "tag-ui", "name": "UI", "color": "#ec4899" },
      { "id": "tag-api", "name": "接口", "color": "#3b82f6" },
      { "id": "tag-bug", "name": "Bug", "color": "#ef4444" }
    ]
  }
}
```

**步骤 3：使用定制模板**

下次创建看板时，选择这个自定义模板即可。

---

#### 示例：基于「每日站会」创建团队定制 Prompt

**步骤 1：复制 Prompt 模板**

在模板管理页面找到「每日站会」，点击「复制」。

**步骤 2：编辑副本**

```json
{
  "meta": {
    "name": "每日站会 (研发团队版)",
    "description": "包含昨日工时记录的站会模板"
  },
  "content": {
    "text": "帮我在看板 '{{board.title}}' 中生成每日站会卡片：\n\n1. 昨日完成（附实际工时）\n2. 今日计划（附预估工时）\n3. 阻塞项（需 @{{user.name}} 协助的请标注）\n\n当前看板有 {{board.laneCount}} 个列表，请根据列状态推断进展。",
    "model": "deepseek-chat",
    "autoSend": false
  }
}
```

> 这个模板使用了变量系统（见第六章），会自动替换当前看板名称、用户姓名等信息。

## 六、模板变量系统

### 6.1 变量语法

模板内容中支持使用双大括号语法插入动态变量：

```
{{variableName}}           → 基础变量
{{nested.property}}        → 嵌套属性
{{variable|fallback}}      → 带默认值
```

### 6.2 可用变量表

| 变量 | 说明 | 示例输出 |
|------|------|----------|
| `{{board.id}}` | 当前看板 ID | `board-abc123` |
| `{{board.title}}` | 当前看板标题 | `Sprint 1` |
| `{{lane.id}}` | 当前列表 ID | `lane-xyz789` |
| `{{lane.title}}` | 当前列表标题 | `进行中` |
| `{{card.id}}` | 当前卡片 ID | `card-def456` |
| `{{card.title}}` | 当前卡片标题 | `登录功能开发` |
| `{{date.now}}` | 当前时间 ISO 格式 | `2026-05-12T08:30:00.000Z` |
| `{{date.today}}` | 今日日期 | `2026-05-12` |
| `{{date.tomorrow}}` | 明日日期 | `2026-05-13` |
| `{{user.name}}` | 当前用户名 | `张三` |

### 6.3 变量使用示例

#### 卡片模板中使用变量

```json
{
  "title": "【Bug】{{card.title}}",
  "description": "报告人：{{user.name}}\n日期：{{date.today}}\n\n**问题描述**："
}
```

创建卡片时如果：
- 用户输入标题：`登录页面崩溃`
- 当前用户：`张三`
- 今日日期：`2026-05-12`

最终生成：

```
标题：【Bug】登录页面崩溃
描述：
报告人：张三
日期：2026-05-12

**问题描述**：
```

#### Prompt 模板中使用变量

```
当前看板：{{board.title}}
列表：{{lane.title}}
用户：{{user.name|开发者}}

请帮我在「{{lane.title}}」中生成今天的任务建议。
```

如果 `user.name` 不存在，则使用默认值 `"开发者"`。

## 七、创建与管理模板

### 7.1 打开模板管理器

路径：**看板页面 → 工具栏 → 模板库**

或直接在浏览器访问带模板管理器的状态（通过设置菜单）。

### 7.2 界面说明

```
┌─────────────────────────────────────────┐
│  模板类型        │  模板库                │
│                  │  ┌────────┐ ┌──────┐  │
│  ○ 全部          │  │搜索... │ │导入 │  │
│  ● 看板          │  └────────┘ │导出 │  │
│  ○ 卡片          │             │新建 │  │
│  ○ 列表          │  ┌─────────────────┐  │
│  ○ 规则          │  │ 基础看板  [内置]  │  │
│  ○ 提示词        │  │ 个人任务管理...   │  │
│                  │  │ [使用] [复制]     │  │
│                  │  └─────────────────┘  │
│                  │  ┌─────────────────┐  │
│                  │  │ Scrum Sprint    │  │
│                  │  │ 敏捷开发...      │  │
│                  │  │ [使用] [复制]     │  │
│                  │  └─────────────────┘  │
└─────────────────────────────────────────┘
```

### 7.3 创建新模板

1. 点击右上角 **「新建模板」**
2. 选择模板类型
3. 填写元数据（名称、描述、标签、图标）
4. 填写内容（根据类型不同，表单字段不同）
5. 点击保存

**示例：创建一个「周复盘」列表模板**

```
类型：列表
名称：周复盘
描述：每周五下午用于回顾本周工作的列表
标签：管理
图标：FileText

内容：
  标题：本周复盘
  初始卡片：
    - 标题：本周完成项
      描述：- [ ] 回顾本周所有已完成的卡片
    - 标题：下周重点
      描述：- [ ] 确定下周最高优先级的 3 件事
    - 标题：阻塞与风险
      描述：- [ ] 记录本周未解决的阻塞项
```

---

### 7.4 编辑与删除

- **自定义模板**：显示「编辑」和「删除」按钮
- **内置模板**：只显示「复制」按钮（不可直接编辑）

> ⚠️ 删除模板是永久操作，不会删除已应用该模板创建的业务数据。

## 八、导入与导出

### 8.1 导出模板

**操作：**

1. 在模板管理页面选择要导出的模板（通过左侧类型筛选）
2. 点击 **「导出」** 按钮
3. 浏览器自动下载 `templates-export-YYYY-MM-DD.json`

**导出文件示例：**

```json
{
  "version": 1,
  "exportedAt": "2026-05-12T08:30:00.000Z",
  "templates": [
    {
      "meta": {
        "id": "board-scrum",
        "type": "board",
        "name": "Scrum Sprint",
        "builtin": false
      },
      "content": {
        "lanes": [
          { "title": "Backlog" },
          { "title": "Todo" }
        ]
      }
    }
  ]
}
```

---

### 8.2 导入模板

**操作：**

1. 点击 **「导入」** 按钮
2. 选择 `.json` 文件
3. 系统自动解析并导入

**导入规则：**

| 情况 | 处理方式 |
|------|----------|
| ID 冲突且原模板非内置 | 覆盖原模板 |
| ID 冲突且原模板是内置 | 跳过（保护内置模板） |
| ID 不冲突 | 新增模板 |

**示例：团队共享模板包**

假设你的团队维护一个 `team-templates.json` 文件：

```json
{
  "version": 1,
  "exportedAt": "2026-05-12T00:00:00.000Z",
  "templates": [
    {
      "meta": {
        "type": "board",
        "name": "迭代冲刺（团队版）",
        "description": "含 Code Review 和 QA 环节的迭代流程",
        "tags": ["团队", "敏捷"],
        "icon": "Rocket",
        "scope": "global",
        "builtin": false
      },
      "content": {
        "lanes": [
          { "title": "需求池" },
          { "title": "开发中" },
          { "title": "Code Review" },
          { "title": "QA 测试" },
          { "title": "已上线" }
        ],
        "tags": [
          { "id": "p0", "name": "P0", "color": "#ef4444" },
          { "id": "p1", "name": "P1", "color": "#f59e0b" },
          { "id": "p2", "name": "P2", "color": "#3b82f6" }
        ]
      }
    },
    {
      "meta": {
        "type": "card",
        "name": "技术方案",
        "description": "技术方案评审卡片模板",
        "tags": ["技术"],
        "icon": "FileText",
        "scope": "global",
        "builtin": false
      },
      "content": {
        "title": "【技术方案】",
        "description": "**背景**：\n\n**目标**：\n\n**方案对比**：\n| 方案 | 优点 | 缺点 |\n|------|------|------|\n| A | | |\n| B | | |\n\n**决策**：\n\n**排期**：",
        "tags": ["技术"]
      }
    }
  ]
}
```

新成员入职时直接导入此文件，即可使用团队统一的模板体系。

## 九、常见问题

### Q1：内置模板可以删除吗？

**不可以。** 内置模板受系统保护，只能复制后编辑副本。如果你不希望看到某些内置模板，可以在模板选择弹窗中通过搜索过滤。

### Q2：复制后的模板和原模板有什么关系？

**完全独立。** 复制操作是"深拷贝"，新模板有独立的 ID 和内容。后续修改原内置模板或自定义副本，互不影响。

### Q3：模板变量在所有类型中都有效吗？

**目前主要用于 `prompt` 和 `card` 类型。** 看板和列表模板在创建时由系统提供上下文变量，但用户手动编辑模板内容时也可以嵌入变量（在支持解析的地方生效）。

### Q4：自定义模板可以设为非全局吗？

**可以。** 创建或编辑模板时，将 `scope` 设为 `board` 并指定 `boardId`，即可让该模板只在特定看板下可见。适用于团队不想共享的私有工作流。

### Q5：模板数据存储在哪里？

模板文件存储在 `data/templates/` 目录下，每个模板一个 `.json` 文件，索引保存在 `data/templates/templates.json` 中。这些数据会随项目一起备份和迁移。

### Q6：如何重置模板系统？

如需重新初始化内置模板：

1. 停止开发服务器
2. 删除 `data/templates/` 目录下所有文件（保留空目录）
3. 删除 `.migration-complete` 标记文件（如果存在）
4. 重启服务器，系统会自动重新创建内置模板
